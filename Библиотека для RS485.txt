В этой библиотеке Modbus RTU реализован с помощью разделения на транспортный и портовый уровни. Инициализация начинается с создания транспорта в mbm_rtu_transp_create, где настраиваются callback-функции для отправки и получения данных, создаётся UART-порт и таймер для соблюдения тайминга T35 (3.5 символа), критически важного для протокола RTU.
При отправке в mbm_rtu_transp_send к кадру добавляется адрес ведомого устройства и CRC16, рассчитанная с использованием табличного метода. При приёме в mbm_rtu_transp_receive проверяется длина кадра и корректность CRC. Работа с UART построена на событиях FreeRTOS, где задача mb_port_ser_task обрабатывает данные и таймауты. Вся реализация строго следует спецификации Modbus RTU, обеспечивая надёжную связь между мастером и ведомыми устройствами.

Вот список операций, выполняемых библиотекой при работе с функциями 03H, 06H и 10H в контексте Modbus RTU:
Общие этапы для всех функций:
Прием кадра: Побайтовый прием данных в кольцевой буфер через прерывание UART.
Детектирование конца кадра: Срабатывание таймера по истечении времени T3.5 (пауза между кадрами).
Валидация кадра: Проверка адреса ведомого, длины пакета и расчет/сравнение контрольной суммы CRC-16.
Парсинг запроса: Извлечение кода функции (03H, 06H, 10H), начального адреса регистра и количества регистров/значений.
Проверка доступа: Сверка адресов с таблицей регистров (mbm_parameter_table_t) на предмет существования и прав доступа (чтение/запись).
Специфика функции 03H (Чтение регистров удержания / Мониторинг параметров d-xx):
6. Чтение данных: Последовательное чтение значений из структуры параметров или массива регистров, начиная с указанного адреса.
7. Формирование ответа: Создание заголовка ответа (адрес + код функции + байтовый счетчик).
8. Упаковка данных: Преобразование 16-битных значений параметров в формат Big Endian (старший байт первым) и запись в буфер ответа.
9. Расчет и добавление CRC: Вычисление контрольной суммы для всего кадра ответа.
10. Отправка: Передача готового кадра в линию связи.
Специфика функции 06H (Запись одного регистра / Настройка параметра P_x.xx):
11. Извлечение значения: Чтение 16-битного значения для записи из тела запроса.
12. Валидация диапазона: Проверка, находится ли записываемое значение в допустимых пределах для данного параметра (если реализовано в библиотеке).
13. Запись в память: Атомарная запись нового значения в соответствующую ячейку структуры параметров или энергонезависимую память (EEPROM/Flash), если требуется сохранение.
14. Формирование эхо-ответа: Копирование исходного запроса (адрес, функция, адрес регистра, записанное значение) в буфер ответа (согласно стандарту, ответ должен полностью совпадать с запросом).
15. Расчет и добавление CRC: Вычисление контрольной суммы для кадра подтверждения.
16. Отправка: Передача кадра подтверждения мастеру.
Специфика функции 10H (Запись нескольких регистров / Пакетная настройка):
17. Проверка количества: Валидация количества регистров для записи (обычно от 1 до 16 или больше, в зависимости от реализации).
18. Циклическая запись: Итерация по списку значений из запроса и последовательная запись их в соответствующие адреса параметров (например, одновременная запись в P0.03 и P0.04).
19. Применение настроек: Вызов колбэков или функций применения изменений, если новые значения требуют перенастройки оборудования (например, изменение частоты).
20. Формирование сокращенного ответа: Создание кадра ответа, содержащего только адрес, код функции (10H), начальный адрес и количество записанных регистров (без самих данных).
21. Расчет и добавление CRC: Вычисление контрольной суммы для кадра подтверждения.
22. Отправка: Передача кадра подтверждения мастеру.
Обработка ошибок (для всех функций):
23. Генерация исключения: Если адрес неверен, данные выходят за диапазон или запись запрещена, формирование кадра ошибки (код функции + 0x80 + код исключения).
24. Отправка ошибки: Передача кадра с исключением мастеру.

**Приложение 1. Протокол связи MINI (MODBUS)**  
(полный собранный документ с **всеми таблицами** и описанием работы протокола)

### 1. Общие сведения о протоколе
Протокол связи MINI основан на **стандарте MODBUS**.  
Главный компьютер (хост) находится в режиме **активного опроса**, ведомое устройство (преобразователь частоты) — в режиме **пассивного ответа**.

**Важное примечание:**  
Не на все запросы хоста ведомое устройство отвечает. При использовании **широковещательного адреса** (broadcast) ведомое устройство **не отвечает**.

### 2. Основные параметры связи

| Параметр                  | Описание                                      | Значение / диапазон          |
|---------------------------|-----------------------------------------------|------------------------------|
| Адрес устройства          | Уникальный адрес в сети RS-485                | 01～31 (десятичный)          |
| Заводская установка       | —                                             | 01                           |
| Скорость передачи         | См. группу параметров P8                      | Выбирается в P8.01           |
| Формат данных             | 3 возможных формата (см. ниже)                | 8,N,2 / 8,E,1 / 8,O,1       |
| Таймаут связи             | Время ожидания ответа                         | См. группу P8                |

### 3. Форматы данных (битовый поток)

▶ **Формат данных 8, N, 2**  
▶ **Формат данных 8, E, 1**  
▶ **Формат данных 8, O, 1**

### 4. Структура кадра данных (режим RTU)

**Полная структура кадра:**

**10 мс интервал + адрес ведомого + функциональный код + данные + CRC CHK + 10 мс интервал**

| Поле            | Описание                                      | Длина          |
|-----------------|-----------------------------------------------|----------------|
| STX             | Пауза без сигнала ≥ 10 мс                    | —              |
| Address         | Адрес ведомого (8 бит)                        | 1 байт         |
| Function        | Функциональный код (8 бит)                    | 1 байт         |
| DATA(n-1)       | Данные (n×8 бит, n ≤ 2)                       | несколько байт |
| CRC CHK Low     | CRC (младший байт)                            | 1 байт         |
| CRC CHK High    | CRC (старший байт)                            | 1 байт         |
| END             | Пауза без сигнала ≥ 10 мс                    | —              |

### 5. Поддерживаемые функциональные коды

| Код   | Название                                      | Широковещание | Макс. количество |
|-------|-----------------------------------------------|---------------|------------------|
| 03H   | Чтение содержимого регистров                  | Нет           | 5 адресов        |
| 06H   | Запись одного слова в регистр                 | Да            | 1 параметр       |
| 10H   | Запись нескольких слов подряд                 | Да            | 2 параметра      |

#### 5.1. 03H — Чтение регистров

**Запрос:**  
`Адрес + 03H + адрес регистра (2 байта) + количество регистров N (2 байта) + CRC (2 байта)`

**Ответ:**  
`Адрес + 03H + количество байт данных (N×2) + данные + CRC (2 байта)`

**Пример:** Чтение P0.06 (адрес 0006H) у устройства 01H  
Запрос: `01H 03H 00H 06H 00H 01H 64H 0BH`  
Ответ: `01H 03H 02H 10H 88H B5H E2H`

#### 5.2. 06H — Запись одного слова

**Запрос / Ответ** (совпадают):  
`Адрес + 06H + адрес регистра (2 байта) + данные (2 байта) + CRC (2 байта)`

**Пример:** Запись 5000 (1388H) в P0.06 устройства 01H  
Запрос / Ответ: `01 06 00 06 13 88 64 9D`

#### 5.3. 10H — Запись нескольких слов

**Запрос:**  
`Адрес + 10H + начальный адрес + количество регистров N + количество байт (N×2) + данные1 + данные2 + CRC`

**Ответ:**  
`Адрес + 10H + начальный адрес + количество регистров N + CRC`

**Пример:** Одновременная запись P0.14=10.0 (0064H) и P0.15=8.0 (0050H)  
Запрос: `01 10 00 0E 00 02 04 00 64 00 50 33 C0`  
Ответ: `01 10 00 0E 00 02 20 0B`

Двоичный вид: **0 0000000 0 0 1 0 0 0 0 1**

|-------|----------|------------------------------------------------|
| Bit0  | 1        | Напряжение нормальное                          |
| Bit2  | 0        | Выходная фазовая последовательность — прямая  |

### 8. Проверка CRC (RTU режим)

**Алгоритм расчёта CRC:**

1. CRC-регистр = FFFFH  
2. Первый байт XOR с младшим байтом CRC-регистра  
3. Сдвиг CRC-регистра вправо на 1 бит (старший бит = 0)  
4. Если перед сдвигом младший бит был 1 → XOR с 0xA001  
5. Повторить 8 раз для каждого байта  
6. Младший байт CRC передаётся первым.

**Реализация на C (точно по оригиналу):**

```c
unsigned int crc_chk(unsigned char *data, unsigned char length)
{
    int j;
    unsigned int reg_crc = 0xffff;
    while (length--)
    {
        reg_crc ^= *data++;
        for (j = 0; j < 8; j++)
        {
            if (reg_crc & 0x01)
                reg_crc = (reg_crc >> 1) ^ 0xA001;
            else
                reg_crc = reg_crc >> 1;
        }
    }
    return reg_crc;   // Возврат значения CRC-регистра
}
```
